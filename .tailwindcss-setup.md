## üöÄ How Consumers Use This Library

### 1. Install the package:
```bash
yarn add @jim850221/react-components-starter
```

### 2. Import styles in their main CSS file:
```css
@import '@jim850221/react-components-starter/styles.css';
```

### 3. Configure their `tailwind.config.js`:
```js
export default {
  content: [
    './src/**/*.{js,ts,jsx,tsx}',
    './node_modules/@jim850221/react-components-starter/dist/**/*.{js,ts,jsx,tsx}',
  ],
  presets: [
    require('@jim850221/react-components-starter/tailwind.config')
  ],
}
```

### 4. Use components:
```tsx
import { MyButton } from '@jim850221/react-components-starter'

<MyButton variant="primary" size="lg">Click me</MyButton>
```

## üé® Adding More Components

When creating new components, follow this pattern:

```tsx
import { tv, type VariantProps } from 'tailwind-variants'

const myComponent = tv({
  base: 'base-classes-here',
  variants: {
    variant: {
      option1: 'classes-for-option1',
      option2: 'classes-for-option2',
    },
  },
  defaultVariants: {
    variant: 'option1',
  },
})

type MyComponentVariants = VariantProps<typeof myComponent>

interface MyComponentProps extends MyComponentVariants {
  // additional props
}

export const MyComponent: React.FC<MyComponentProps> = ({ variant, ...props }) => {
  return <div className={myComponent({ variant })} {...props} />
}
```

## ‚ö†Ô∏è Important Notes

1. **Tailwind as Peer Dependency**: This prevents bundling Tailwind twice when consumers install your library
2. **Export Config**: Consumers can extend your Tailwind config as a preset
3. **Export Styles**: Consumers must import your styles.css to get base styles
4. **Content Paths**: Consumers must add your library's path to their Tailwind content array

## üîß Troubleshooting

If styles aren't applying in consuming projects:
1. Ensure they imported `@jim850221/react-components-starter/styles.css`
2. Check their `tailwind.config.js` includes your library in `content` array
3. Verify they have `tailwindcss` installed as a peer dependency
4. Make sure they're processing CSS with PostCSS + Tailwind plugin
